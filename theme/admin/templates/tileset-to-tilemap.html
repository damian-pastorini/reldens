      How to Use - Step by Step

      Step 1 - Upload your tileset

      Select one or more PNG tileset files. Set the tile width and height (use the preset buttons for common sizes). Adjust spacing and margin if your tileset has gaps between tiles. The background color is auto-detected from the image edges - correct it if needed. Click Analyze to start.

      Step 2 - Wait for detection

      The server scans every tile for non-background pixels and groups connected tiles into clusters. If AI providers are active, each cluster is sent to AI for sub-element detection and naming. Progress is shown in real time. This may take a minute for large tilesets.

      Step 3 - Review detected elements

      The canvas shows your tileset with colored overlays for each element. The legend on the right lists every detected item.

      • Elements (solid border, locked by default): individual objects identified by pixel analysis or AI.
      • Clusters (dashed amber border, unlocked): groups that may contain multiple objects and need further processing.
      • Faint gray tiles: tiles that passed the fill check but had too little variance to be grouped (shown for reference only).

      Step 4 - Canvas view modes

      By default the canvas is in View all mode: all elements are visible and you can right-click any tile to remove it from its owner element.

      • View selected button: switches to view-selected mode. All elements except the selected one are dimmed. Click again (View all) to return to view-all mode.
      • Highlight all button: toggles colored overlays for all elements while one is selected. Click again (Hide others) to restore the dimmed view.

      Step 5 - Canvas shortcuts

      • Left-click a tile: add the tile to the selected element (or remove it if it is already in the active layer). Requires an element to be selected.
      • Left-click drag: bulk-add or remove tiles based on the first tile action.
      • Right-click a tile: remove the tile from its owner element. Works in view-all mode (removes from any element) and when an element is selected (removes from that element).
      • Right-click drag: keep removing tiles continuously while holding the mouse button.
      • Shift+right-click drag: draw a rectangular area selection - all tiles inside the rectangle are removed when you release the mouse button. A red overlay shows the selection while dragging.
      • Shift+left-click: select the element that owns the clicked tile.
      • Ctrl+scroll: zoom canvas in or out (0.25× to 4×). Use the Reset zoom button to return to 100%.
      • Delete key: remove the currently selected element entirely.

      Step 6 - Edit element properties

      Click an element row to expand its controls:

      • Name: rename using kebab-case format, e.g. tree-001.
      • Lock (padlock): locked elements are skipped by Detect All and Name All. Lock elements you are satisfied with.
      • Delete (trash): remove the element entirely. Same as pressing the Delete key on the keyboard.
      • Layer type: choose what layer new tiles will be added to (below-player, collisions, over-player, collisions+over).
      • Quantity: how many copies to place in the generated map.
      • Free space: tile padding around the element in the generated map.

      Step 7 - Cluster actions

      • Detect (per element): send this cluster to AI for sub-element detection. Replaces the cluster with its sub-elements.
      • Split into tiles: convert each tile in the cluster into a separate single-tile element.
      • Convert to element: treat the entire cluster as one approved element.

      Step 8 - Bulk actions

      Check the checkbox on any element row to include it in bulk operations. Buttons appear in the tileset header:

      • Convert all: converts all checked clusters to elements.
      • Toggle lock: toggles the locked state of all checked elements.
      • Detect AI: runs AI sub-detection on all checked clusters (requires AI provider).
      • Name AI: runs AI naming on all checked non-cluster elements (requires AI provider).

      Step 9 - AI bulk actions

      • Detect All: runs AI sub-detection on every unlocked cluster across all tilesets.
      • Name All: runs AI naming on every unlocked non-cluster element. Locked elements are skipped.
      • Progress is shown next to the buttons. After Name All completes, a summary shows how many were renamed and how many were skipped.

      Step 10 - Generate files

      When all elements are named and configured, click Generate files. The server creates Tiled-compatible JSON files for each element, an annotated tileset PNG, a composite map, and a Reldens map-generator config. Download links appear in the Results section.

      Tips

      • Lock an element after you are happy with it - this protects it from AI bulk operations.
      • Add element creates a blank element you can populate by clicking tiles on the canvas.
      • Previous sessions are listed at the bottom of the upload form - click to expand and download files again.
      • All element names must match the pattern word-001 (lowercase letters, optional hyphens, numeric suffix) before Generate is enabled.
